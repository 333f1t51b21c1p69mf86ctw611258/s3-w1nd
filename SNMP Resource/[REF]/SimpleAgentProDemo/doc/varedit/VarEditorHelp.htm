Menu Bar      

            File Menu:

The File menu contains the following items:

·        Open: If there is already an open variable file, the user will be asked if it should be saved and closed. Consequently or if there is no open variable file, a “varible file selection dialog” will come up and the user can type in the project that should be opened.

·        Save: The variable file will be saved with the same name that it was opened with. If this was a new “untitled” variable file, the user will be asked to provide a name to save it as.

·        Save As : The user will be asked to provide a name to save the variable file as.

·        Exit : Will perform the function of the “Save” command and then terminate the Var File Editor session.

 

EditMenu :

The Edit menu contains the following items:

·        All the options that are available in the “Node Popup Menu”

Node Popup Menu

Whenever the right button is clicked on a node, a context sensitive popup menu pops up. If multiple nodes are      selected, then options that make sense for multiple nodes appear enabled. Rest of the options are disabled in the

 menu.

 

(Note:  The “cut” option would remain active even when nodes of different type are selected. Table nodes and Entry nodes should be treated similarly. Options available for Table Nodes apply to Entry nodes too.)

Index

Address Node

             (non-leaf-nodes excluding the Table Nodes)

 Denoted as ‘ N’ on the node

Available Operations: Create Instance

Whenever the right button is clicked on a node, a context sensitive Popup Menu pops up.        

 

Create Instance: Whenever new instance is to be created this option is used .

 Newly created instance will have the OID as a name , and will be added as a scalar node at appropriate position in the tree.

 

If only  node is selected, then Create Instance menu item is enabled.

If multiple nodes are selected then all menu items on Popup Menu and EditMenu  are disabled.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Index

Create Instance

Availability: Only for Address Nodes.

Functionality: Creates new node

 

                             

 

The “Create Instance ” dialog box pops up where the user can assign an OID to this node.   The OID of the new node will have the default prefix as the OID of the address node.

 

User can give number , which should have only one dot or may not have a dot   .Then It will be added at appropriate position.

There after the “Create/Modify Value” dialog box pops up where the user can assign a value. The user has to assign a value. There is no cancel operation on this dialog.

 

Note : If it is a duplicate OID then  it is not added in the tree. “Duplicate instance ”   warning is flashed on the screen.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Index

            Dynamic Row Template

 

Availability : Only for Table Nodes.

Functionality:

Brings up the “Dynamic Row Template” dialog corresponding to this table. If there is an existing “dynamic row template” for the table.

 

User can set required /not required status for the column.

Also can change the value of column by clicking on value ,which

invokes “Create/modify” dialog box.

 

                       

 

                        Type: it can be rowstatus, rmonstatus, or newinstance

                        Column: which column to modify

                        Value: any integer value

 

Note :  A check mark is placed ahead of this menu item in the popup menu if dynamic row template is present for the table.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Index

Create Modify Instance

 

Availability: Only for Scalar, Instance, Table and Column Nodes.  All the selected               

nodes should be of same type .

Functionality: For scalar nodes, the “Create/Modify Value” dialog box pops up.

For table node, the “Table View” dialog pops up and for column nodes, the “Column View” dialog pops up.

 

 

                       

 

            Type: Is a combo list box with the following SNMP data type options :

     Counter, Gauge, IpAddress, NetworkAddress, Integer, OctetString, ObjectID, Bits, Counter64.

            Access: specifies which access

Value Information:  Format is a combo list box to specify the format for the value (example fixed, randomup …)

 

Note:  For columnar nodes which are instance for a table node then , no modification  

                        of  information is allowed.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Index        

Table View Dialog

 

                Availability:  For Table Nodes

Functionality:

This dialog pops up when the user wishes to create/modify instance in a Table. The instance column shows the key for this row (made up of  concatenation of several of the column fields). Whenever user double clicks on a cell, the “Create/Modify Value” dialog pops up. User can use this table like a spreadsheet to add/copy/delete rows. The title of the dialog box shows the “OID” for the table.

 

The user can  select the “Column” header and set the value type for the whole column.

                       

                                   

 

The toolbar at the top will have the following items :

·        Save: Upon execution of this command, the table will be save at the node from which the dialog was invoked.

·        Delete: If there are rows selected in the table, they will be deleted.

·        Add: New rows are added. The row will be shown in the table initially at the bottom. As the “instance” name get generated, it will be shown where it fits lexicographically.

·        Exit : Will perform the function of the “Save” command and then popdown this dialog.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Index

Column View Dialog

Availability:  For Column Nodes

Functionality:  To modify columnar node information.

 

 

                       

 

This dialog pops up when the user wishes to create/modify instance for a Column. The instance column shows the key for this row.

Whenever user double clicks on a cell, the “Create/Modify Value” dialog pops up.

 

User can NOT delete or add a row. The title of the dialog box  shows the “OID” for the column.

 

The user can select the “Column” header and set the value type for the whole column.

 

The toolbar at the top will have the following items :

Save: Upon execution of this command, the column will be save at the node from which the dialog was invoked.

Exit : Will perform the function of the “Save” command and then popdown this dialog.

Copy:

 

Availability : Only for Scalar, Instance, Table and Column Nodes.

Same as the Cut option except that the nodes are not deleted from the variable tree.

Functionality: Copies the values of  node.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Index

Cut:

 

Availability : Only for Scalar, Instance, Table and Column Nodes.

The selected instances are deleted from the variable tree.

If only one SCALAR node was selected, the cut value is kept on the clipboard and can be pasted later on to a node of matching type. The node name is not saved on the clipboard, only the attributes (type of node –read, write … and the valuetype) corresponding to this node are saved.  The cut and paste is now limited only to scalars in this version.

 

If the deleted node existed in both in the var-file and MIB, then the appearance will change.  If it existed only in var-file or only in MIB, then it will be physically removed from the tree.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Index

Paste:

 

Availability : Only for Scalar and Instance Nodes.

Functionality : This option is available if there is anything on the clipboard.

If a scalar/instance node A exists on the clipboard and the pasting is occuring on a scalar/instance node B, the node B will now have the Value Type, Access Information and the Value Information of node A. Note that replicating the OID does not make sense in this scenario.

 

Note:  NO OTHER TYPE OF CUT AND PASTE SOURCE/DESTINATION         SCENARIO IS  BEING SUPPORTED FOR THE FIRST RELEASE.

Column Node

             (Children of Entry Nodes – indirectly children of table nodes.)

A Column node shows how many rows are present for that column. This would be the same as the columns in the table to which this column belongs.

Denoted as ‘C’ on the node

Available Operations : Create Modify Instance , Cut ,Copy ,Paste

Whenever the right button is clicked on a node, a context sensitive PopupMenu pops up.

If only one column node is selected, then Create Modify Instance, Cut ,Copy ,Paste menu items are enabled.

 

 Create Modify Instance  for column node invokes Column View Dialog.

It allows to modify column node values.

 

If multiple column nodes are selected then  if  they are of same type then only Cut ,Copy ,Paste on menu items PopupMenu and EditMenu  are enable . Otherwise all are disabled.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Index

Scalar Node

 

Denoted as ‘S’ on the node

Available Operations : Create Modify Instance , Cut ,Copy ,Paste

Functionality: Whenever the right button is clicked on a node, a context sensitive PopupMenu pops up.

If only one scalar node is selected, then Create Modify Instance, Cut ,Copy ,Paste menu items are enabled.

If multiple nodes are selected then  if  they are of same type then only Cut ,Copy ,Paste on menu items PopupMenu and EditMenu  are enable . Otherwise all are disabled.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Index

Table Node

Denoted as ‘N’ on the node

 

Available Operations :  Dynamic Row Template, Create Modify Instance , Cut ,Copy and Paste

Whenever the right button is clicked on a node, a context sensitive PopupMenu pops up.

If only one table node is selected, then Create Modify Instance, Cut ,Copy ,Paste and Dynamic Row Template menu items are enabled.

For table node,  Create Modify Instance invokes Table View Dialog.

If multiple table nodes are selected then only Cut ,Copy ,Paste on menu items PopupMenu and Edit Menu  are enable . Otherwise all are disabled.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Index

Entry Node

 

Denoted as ‘N’ on the node below Table Node :

Available Operations :  Create Modify Instance, Cut ,Copy ,Paste and

                                    Dynamic Row Template

Whenever the right button is clicked on a node, a context sensitive PopupMenu pops up.

If only one Entry node  is selected, or a single Entry node as well as it’s parent

 

Table Node are selected  then Create Modify Instance,  Cut ,Copy ,Paste and Dynamic Row Template menu items are enabled.

For Entry node,  :  Create Modify Instance  invokes Table View Dialog.

 

Note:  If multiple entry nodes or entry node along with table node belonging to other 

entry node are selected then, only Cut ,Copy ,Paste on menu items PopupMenu and EditMenu  are enable . Otherwise all are disabled.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Index

            Instance Node

 

These nodes are leaf nodes for variables defined in the “.var” file but do not exist in the MIB file. The behavior is exactly like the scalar nodes.

 

Available Operations :

Whenever the right button is clicked on a node, a context sensitive PopupMenu pops up.

If only one instance node is selected, then Create Modify Instance , Cut ,Copy ,Paste menu items are enabled.

 

If multiple nodes are selected then  if  they are of same type then only Cut ,Copy ,Paste on menu items PopupMenu and EditMenu  are enable . Otherwise all are disabled.
